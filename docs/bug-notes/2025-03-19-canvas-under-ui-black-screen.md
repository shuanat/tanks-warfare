# Canvas перекрывает меню — чёрный экран в лобби

**Дата:** 2025-03-19  
**Статус:** исправлено

## Симптом

В лобби/меню вместо формы — сплошной чёрный фон; UI не виден.

## Ожидание

Меню и лобби поверх игрового фона, canvas не мешает до старта боя (или не перекрывает фиксированный UI).

## Причина

`#gameCanvas` шёл в DOM после оверлеев, без `z-index`. При `display: block` и полном размере canvas визуально оказывался **выше** меню (`z-index` не задан = auto / один контекст наложения), перекрывая его чёрной заливкой кадра.

## Фикс

- `client/src/styles/main.css`: у `#gameCanvas` — `position: fixed`, `z-index: 1`; у `#menu` / `#lobby` — высокий `z-index` (например 100), чтобы UI был сверху.

## Проверка

Открыть `npm run dev:client`, не начиная бой: видны поля меню и кнопки.
