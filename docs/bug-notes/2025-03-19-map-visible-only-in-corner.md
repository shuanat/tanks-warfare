# Игровой мир виден только в части экрана (часто правый нижний сектор)

**Дата:** 2025-03-19  
**Статус:** исправлено

## Симптом

Сетка/карта занимают лишь фрагмент окна (например ~25–30% справа снизу), остальное чёрное. HTML-HUD по краям выглядит нормально.

## Ожидание

Карта визуально заполняет область canvas; нет больших «пустых» зон вне тайлов карты при обычном спавне.

## Причина

1. **Камера:** `camX`/`camY` следуют за танком у **левого/верхнего** края карты. Точка на экране переводится в мировые координаты; для пикселей, попадающих в область **x < 0 или y < 0** в мире, фон не рисуется (нет `fillRect` карты) — остаётся чёрный клир. Визуально — «карта в углу».
2. Дополнительно на части конфигураций полезно синхронизировать **DPR** и порядок загрузки CSS/JS; основной визуальный «угол» для спавна слева даёт п.1.

## Фикс

- `client/src/game/gameClient.js`: **`clampCamera`** — после расчёта камеры от танка и мыши ограничить `camX`/`camY`, чтобы видимый прямоугольник оставался в `[0, mapW] × [0, mapH]` (с половиной ширины/высоты вида в мировых единицах).
- Ресайз canvas: логические `innerWidth`/`innerHeight`, буфер × `devicePixelRatio`, `ctx.setTransform(dpr,…)`, явные `canvas.style.width/height`.
- `client/src/main.js`: импорт стилей **до** `gameClient`.
- Мышь: координаты относительно canvas (`getBoundingClientRect`).

## Проверка

Спавн команды 1 (левый сектор): весь видимый canvas заполнен полем/сеткой без огромного чёрного L-образного фона.
