# Заметки по багам и фиксам

Короткие **мини-репорты**: что ломалось, почему (гипотеза / корневая причина), что сделали, где в коде. Нужны, чтобы не терять контекст между сессиями и быстрее разбирать похожие симптомы.

## Как добавлять запись

1. Создай файл **`YYYY-MM-DD-краткий-слаг.md`** (дата фикса или первой диагностики, латиница и дефисы в слаге).
2. Скопируй структуру из [`_TEMPLATE.md`](_TEMPLATE.md) и заполни. Достаточно **5–15 строк** на раздел; без романа.
3. В **Симптомах** — что видит игрок или что в консоли. В **Причине** — одна главная, без воды. В **Фиксе** — файлы и суть патча.

При одном PR с несколькими багами можно один файл `YYYY-MM-DD-topic1-topic2.md` или несколько датированных — как удобнее.
